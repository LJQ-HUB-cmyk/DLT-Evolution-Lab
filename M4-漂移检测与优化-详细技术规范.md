# M4 漂移检测与优化详细技术规范（执行版）

## 1. 目标与硬约束
- 目标：建立可量化、可触发、可回滚的模型漂移治理与自动优化体系，避免预测质量静默退化。
- 硬约束：
  - 不允许覆盖 `official` 已发布记录。
  - 漂移判定必须可追溯到可解释子指标与输入快照。
  - 漂移动作（降权/标记/入队优化）必须全量写审计日志。
  - 新模型晋升必须通过“回测门禁 + 稳定性门禁 + 可复现门禁”。

## 2. 范围与非范围
- 范围内：
  - 漂移指标体系、综合漂移分计算、分级告警。

  - Optuna 参数搜索、候选模型注册、冠军晋升流程。

- 范围外：
  - 任务调度框架重构（属于 M5）。

## 5. 漂移指标体系（必须全量实现）

## 5.1 输入定义
- `baseline`：最近一次 `official` 预测记录（同玩法同规则版本）。
- `current`：当前待评估 run（通常为 `predict` 输出）。
- `history_ref`：最近 `N_ref=20` 次实验 run 统计摘要。

## 5.2 子指标定义
- `position_dist_drift`
  - 对每个位置概率分布计算 `Jensen-Shannon Divergence`（JSD）。
  - 公式：`JSD(P||Q)=0.5*KL(P||M)+0.5*KL(Q||M), M=(P+Q)/2`。
  - 各位置取均值，归一化到 `[0,1]`。
- `number_set_drift`
  - 方案1、方案2分别计算前区/后区集合 Jaccard 差异。
  - 差异定义：`1 - |A∩B|/|A∪B|`，按前后区与方案加权平均。
- `structure_drift`
  - 对结构向量比较：
    - 奇偶比、大小比、三区分布、和值、跨度、连号段数、冷热占比。
  - 计算标准化曼哈顿距离：`sum(|x_i-y_i|/range_i)/k`。
- `score_gap_drift`
  - 计算 `ticket_score` 前 K（默认 K=10）均值差与方差差。
  - 用基于 `history_ref` 的 z-score 归一化，截断到 `[0,1]`。
- `plan_overlap_drift`
  - 当前方案1/2与基线方案1/2的重叠稳定性差异。
  - 指标：`overlap_delta = |overlap_current - overlap_ref_mean|/max(overlap_ref_std,eps)`，再 Sigmoid 映射。

## 5.3 综合漂移分
- 综合分：
  - `drift_score = 0.30*position + 0.20*set + 0.20*structure + 0.20*score_gap + 0.10*overlap`
- 漂移等级：
  - `NORMAL`: `drift_score < 0.35`
  - `WARN`: `0.35 <= drift_score < 0.55`
  - `CRITICAL`: `drift_score >= 0.55`
- 单项兜底规则（硬触发）：
  - 若任一子指标 `>= 0.80`，最少判定为 `WARN`。
  - 若两个及以上子指标 `>= 0.75`，直接 `CRITICAL`。

## 6. 漂移触发动作（强约束）

## 6.1 自动降权策略
- 触发条件：`drift_level in {WARN, CRITICAL}`。
- 特征降权：
  - 对高贡献且高漂移关联特征执行 `w' = w * decay_factor`。
  - `decay_factor`：
    - `WARN=0.92`
    - `CRITICAL=0.85`
- 搜索约束收紧：
  - `structure_penalty += delta_penalty`
  - `beam_width` 在 `CRITICAL` 时下降 20%，控制不稳定扩散。

## 6.2 模型信用分（Model Credit）更新
- 每个模型维护 `credit_score in [0,100]`。
- 更新公式：
  - `credit_t = alpha*credit_{t-1} + (1-alpha)*(base_reward - drift_penalty - instability_penalty)`
  - 默认 `alpha=0.8`
  - `base_reward=70`
  - `drift_penalty=40*drift_score`
  - `instability_penalty=15`（若触发可复现告警）
- 状态：
  - `credit >= 70`: `healthy`
  - `50 <= credit < 70`: `watch`
  - `< 50`: `unstable`

## 6.3 自动优化入队
- 入队条件（任意一条成立）：
  - `drift_level=CRITICAL`
  - 连续 3 次 `WARN`
  - `credit_score < 55`
- 入队动作：
  - 调用 `POST /api/optimize`（内部服务调用亦需写日志）。
  - 生成 `optimization_run_id`，状态 `queued -> running -> completed/failed`。

## 7. 优化算法规范（Optuna 必须按此实现）

## 7.1 搜索空间（不得删减）
- 特征相关：
  - `freq_window_main ∈ {30,50,80,120}`
  - `ewma_alpha ∈ [0.10,0.35]`
  - `hot_threshold ∈ [0.5,1.5]`
  - `cold_threshold ∈ [-1.5,-0.5]`
- 搜索相关：
  - `beam_width ∈ [40,180]`
  - `diversity_lambda ∈ [0.05,0.60]`
  - `structure_penalty ∈ [0.10,1.20]`
- 校准相关：
  - `platt_C ∈ [0.1,8.0]`（log）
- 方案2采样相关：
  - `plan2_seed_mix ∈ [0.0,1.0]`
  - `plan2_soft_constraint_scale ∈ [0.6,1.2]`

## 7.2 目标函数（多目标折中）
- 主目标（最大化）：
  - `objective = 0.45*backtest_score + 0.20*stability_score + 0.20*calibration_score + 0.15*diversity_score`
- 子项定义：
  - `backtest_score`：滚动窗口命中代理分（基于奖级代理打分）。
  - `stability_score`：固定 seed 下多窗口方差惩罚后的得分。
  - `calibration_score`：`1 - normalized_brier - normalized_ece`。
  - `diversity_score`：候选票互异性与结构合法率组合。
- 约束惩罚（违反即重罚）：
  - 非法票率 > 0：`-1000`
  - 可复现失败：`-1000`
  - `p95` 超时超预算：`-300`

## 7.3 回测门禁（冠军晋升前）
- 必须执行 Walk-Forward：
  - 窗口：训练 180 期，验证 30 期，步长 10 期，至少 6 折。
- 必须通过门禁：
  - 平均目标分提升 `>= +3%`（相对 champion）
  - 最差折不低于 champion `-2%`
  - 漂移均值不升高超过 `+0.05`
  - 可复现测试 20/20 通过
  - 性能预算：`predict p95 < 3.0s`
