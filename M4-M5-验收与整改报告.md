## 4. 发现问题与整改结果

### 问题 A（已修复）
- 问题：`jobs/register_tasks.ps1` 为模板脚本，未固化 M5 指定时间与轮询策略（09:00、20:30、21:45 起每 5 分钟轮询）。
- 影响：部署后需人工改任务，存在偏离规范和漏配风险。
- 整改：
  - 重写任务注册脚本为“可直接执行版本”。
  - 固化三类任务：
    - `DLT-EvolutionLab-sync-daily-0900`
    - `DLT-EvolutionLab-publish-check-2030`（MON/WED/SAT）
    - `DLT-EvolutionLab-draw-poll-2145-5m`（MON/WED/SAT，`/RI 5`，`/DU 02:00`）
- 结果：调度策略与 M5 文档对齐，不再依赖手工编辑。

### 问题 B（已修复）
- 问题：`draw_poll_job` 原先必须手工传 `--issue --front --back`，不具备“自动轮询官方开奖并链式触发”能力。
- 影响：无法满足 M5 的自动化链路要求（sync -> poll -> postmortem -> optimize -> promotion）。
- 整改：
  - 新增自动轮询入口 `run_draw_poll_job(...)`：
    - 先执行 `sync_job`；
    - 自动选择“未复盘的目标期”；
    - 若已同步到开奖数据，自动链式执行 `run_draw_poll_and_chain(...)`；
    - 若未开奖或无目标期，返回 `skipped` 且不污染数据。
  - `scheduler_service.py` 调整为双模式：
    - 自动模式：`draw_poll_job --trigger schedule`
    - 手工模式：`draw_poll_job --issue ... --front ... --back ...`
  - `publish_check_job` 支持不传 `--issue` 时自动解析下一目标期。
- 结果：调度服务具备全自动运行能力，且保留人工兜底路径。
