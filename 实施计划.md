实施计划

M1 基础工程
前端先起 React + Vite + TypeScript，后端起 FastAPI，补齐根目录工程约定、环境配置、日志、JSON 存储读写层、统一时间/期号工具、基础 API 壳子。验收标准是：页面能打开、后端能启动、GET /api/issues 和 GET /api/models 能返回空结构但类型完整。

M2 数据同步闭环
实现官方页面抓取、HTML 解析、原始快照落 data/raw、标准化结果落 data/normalized/issues.json，并加规则版本表与哈希校验。验收标准是：可从官方源同步历史期次，重复同步不污染数据，异常时进入降级并写 scheduler_logs.json。

M3 首版预测引擎
先做透明特征层、位置评分、基础校准、组合搜索、方案1/方案2出票、正式发布冻结和实验 run 记录。验收标准是：POST /api/publish/{targetIssue} 与 POST /api/predict/{targetIssue} 能稳定产出合法票，且同快照同 seed 完全可复现。

M4 漂移与优化
补 DriftReport、模型信用分、unstable 标记、自动降权、Optuna 搜索、候选模型注册和 walk-forward rolling backtest。验收标准是：连续实验 run 能触发漂移检测，高漂移会自动入队优化，但不会覆盖已发布正式结果。

M5 开奖复盘与自动任务
实现开奖轮询、命中回填、奖级计算、复盘报告、冠军模型晋升，以及 Windows 任务计划启动本地服务和定时任务。验收标准是：开奖后能自动生成 postmortems.json、optimization_runs.json、模型晋升记录和运行日志。

M6 前端体验与测试
完成你要求的双栏 UI、票面视觉、图表、实验日志、复盘详情、模型演进页，并补 API 测试、引擎回归测试、前端布局测试。验收标准是：桌面端左右布局、移动端上下折叠、球样式和图表都稳定可用。

推荐目录细化

apps/api/app/routes: API 路由
apps/api/app/services: sync、publish、predict、postmortem、optimize
apps/api/app/engine: features、calibration、search、drift、backtest
apps/api/app/storage: JSON 仓储层
apps/api/app/models: Pydantic 类型
apps/web/src/pages: 主界面与模型页
apps/web/src/components: 历史栏、票面卡、漂移日志、回测图
jobs: 定时任务入口、任务计划脚本
storage: 你指定的 7 个核心 JSON 文件
